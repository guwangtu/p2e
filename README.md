# P2E: PPG -> ECG 生成实验基线

本仓库用于开展 `PPG -> ECG` 条件生成实验，当前计划统一比较以下 5 条技术路线：

- `RDDM`
- `DDPM`
- `MeanFlow`
- `Flow Matching`
- `iMF (Improved Mean Flow)`

目标是建立一套统一的数据、训练、评估协议，保证不同方法在同一设置下可公平对比。

## 1. 项目目标

- 输入：PPG 一维时序（可单导联或多通道）
- 输出：ECG 一维时序（与输入窗口时间对齐）
- 任务类型：条件生成 / 回归生成
- 核心诉求：在形态保真和生理指标一致性上达到更优结果

## 2. 方法对比总表

| 方法 | 核心思想 | 训练目标 | 采样/推理速度 | 主要优点 | 主要风险 |
|---|---|---|---|---|---|
| RDDM | 基于扩散思想的重建/去噪式生成 | 噪声预测或重建损失 | 慢到中等 | 训练稳定，文献成熟 | 推理步数多，时延高 |
| DDPM | 经典扩散概率模型 | epsilon/v 预测损失 | 慢 | 结果质量通常较高 | 计算开销大 |
| MeanFlow | 均值流场建模 | 流场/速度场匹配 | 快 | 推理步数少 | 对训练细节敏感 |
| Flow Matching | 直接拟合连续流 | 速度场监督 | 快 | 训练目标直接，收敛快 | 易受数据噪声影响 |
| iMF | MeanFlow 的改进版本 | 复合流场目标 + 辅助头 | 快到很快 | 在低 NFE 下性能好 | 迁移到 1D 需仔细适配 |

## 3. 统一实验协议（建议固定）

### 3.1 数据协议

- 统一采样率：例如 `125Hz` 或 `250Hz`
- 统一窗口长度：例如 `T=1024` 或 `T=2048`
- 统一切片策略：固定滑窗长度 + 固定 overlap
- 严格按受试者划分 `train/val/test`，避免泄漏
- 统一归一化策略（推荐先 per-window z-score）

### 3.2 训练协议

- 同一批大小、训练轮数、学习率调度
- 同一模型规模级别（参数量尽量接近）
- 同一随机种子组（建议至少 3 个种子）
- 同一早停/保存策略

### 3.3 评估协议

基础指标（必须）：
- `MAE`
- `RMSE`
- `Pearson r`

形态/频域指标（建议）：
- `DTW`（动态时间规整距离）
- `R-peak timing error`
- `FFT/STFT 频谱误差`

临床相关指标（可选）：
- 心率误差（HR）
- HRV 特征误差（如 SDNN、RMSSD）

## 4. 推荐实验矩阵

| 维度 | 设置 A | 设置 B | 设置 C |
|---|---|---|---|
| 数据长度 T | 1024 | 2048 | 4096 |
| 采样率 | 125Hz | 250Hz | 360Hz |
| 模型骨干 | 1D U-Net | 1D Transformer | Hybrid |
| 损失组合 | L1+L2 | L1+L2+Diff | L1+L2+Diff+Freq |
| 方法 | RDDM | DDPM | MeanFlow/FM/iMF |

> 建议先固定 `T=1024 + L1/L2 + 统一骨干` 完成 5 方法首轮横向对比，再扩展到更长序列和更复杂损失。

## 7. 快速开始

### 7.1 环境

```bash
conda create -n p2e python=3.12
conda activate p2e
pip install -r requirements.txt
```

如果使用现有环境文件：

```bash
conda env create -f environment.yml
```

### 7.2 实验执行建议

1. 先跑一个最小子集（sanity check）
2. 跑 5 方法单 seed 对比
3. 跑多 seed 统计均值与方差
4. 输出统一结果表与可视化波形对比图

## 9. 注意事项

- 不要在不同方法间更换数据切分，否则结果不可比
- 指标方向要统一（`↑` 越大越好，`↓` 越小越好）
- 生成波形必须做可视化抽检，防止“指标好但形态异常”
